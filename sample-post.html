示例文章：如何快速部署 GitHub Pages

这是一个简洁的文章模板，用于在 GitHub Pages 上演示文章发布流程。复制此文件并修改标题与内容，即可作为新文章上传。

第一段示例文字：简要介绍文章内容，说明目的与背景。

步骤一：准备仓库

在 GitHub 上创建仓库并命名为 yourusername.github.io（替换为你的用户名），将页面文件放在仓库根目录并启用 Pages。

步骤二：上传并触发部署

将本文件保存为例如 2025-12-06-demo.html，提交到 main 分支，等待几分钟完成部署后通过 https://yourusername.github.io/2025-12-06-demo.html 访问。

常见问题

  • 404：确保文件已 push 且 Pages 发布分支为包含该文件的分支
  • 被重定向到首页：检查 index.html 是否有 SPA 路由拦截
  • Jekyll 忽略：若使用纯静态文件，建议在根目录添加 .nojekyll

祝你部署顺利！如需我帮你把这个文件直接添加到仓库（创建 PR 或直接 push），告诉我即可。
